Compact Mode — Approach Options

A global toggle to shrink todo cards by hiding descriptions and subtasks

A

Simple Density Toggle

A small icon button in the page header (e.g., rows icon) that toggles between "comfortable" and "compact" views. Compact hides description, subtasks, and tightens padding. State persisted in localStorage.

Pros

  • Simplest to build
  • Familiar pattern (Gmail, Slack)
  • One click, affects all columns

Cons

  • Binary — no middle ground
B

3-Level Density (Comfortable / Compact / Ultra)

A segmented control or cycle button with 3 levels: Comfortable (current), Compact (hide desc + subtasks, tighter spacing), Ultra (single-line per todo — just title + status/priority pills inline).

Pros

  • Ultra mode fits 20+ todos without scrolling
  • Flexible for different screen sizes

Cons

  • More complexity to build and maintain
  • Ultra mode may feel too stripped
C

Compact + Hover Expand

Compact mode hides desc + subtasks, but hovering a card reveals them in a tooltip/popover. Best of both worlds — dense overview with details on demand.

Pros

  • No info loss — just hidden until needed
  • Quick scanning + deep info without mode switching

Cons

  • Hover popovers can feel janky if not tuned
  • Doesn't work on touch/mobile
